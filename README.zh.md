# DO-NOT-FORGET 情感备忘录

一个为“容易被信息流打断、精力起伏明显、但仍渴望留住心动与洞察的人”而生的超轻量工具。它不追求全面记录，而是帮你抓住那些一闪而逝、却真正构成“你”的瞬间。

- 在低能量时段，细腻的体验与感受只会在情境下持续。等到环境变了、会议开完、或睡了一觉，之前的情绪只剩下模糊的标签，激情与感动也渐渐消退。
- 对 ADHD 或易遗忘人群，传统长笔记的启动与组织门槛过高，稍有中断就失去动机；临时的动机与线索常被“下一条消息”“下一段通话”冲走。
- 我们需要一种“低摩擦 + 轻量 + 温和提醒”的方式：即开即写、长度适度限制，帮助抓住情感核心并快速封存，把易散的心流锚定为可追溯的心路档案。

## 这是什么？

- 即开即写：启动就是输入框，无需新建/命名/选择分类。
- 轻度限制：默认 100 字，迫使你提炼情感核心，抑制“写成长文”的拖延。
- 情绪选项：直接从下拉框勾选当前情绪基调，既提醒自我觉察，也方便后期按情绪筛选。
- 温和提醒：最小化到系统托盘，周期性轻提醒，提示你“有感就记”而非“要写长文”。
- 可回望：以时间序列保存，回顾时能看到“我在不同阶段的情绪图景”。
- 纯本地：纯文本 json，离线可用、可自备份，无账号与网络依赖。

## 真实使用场景

- 地铁到站前的 30 秒：手心还残留着扶手的凉，心里突然通透了一句解释自己的话——打开、敲下、保存、合上。
- 与人争执后的走廊里：胸口还在起伏，不求完整复盘，只抓住一句“真正刺痛我的点是什么？”——这句话，足够以后理解自己。
- 夜里醒来的一瞬：没有力气开电脑，更不想分类、起标题——只写“醒来时想到的画面和那种酸涩”，明早再说。
- 做成一个小目标的当下：不是炫耀成果，而是记录“我为什么此刻感到轻松和稳”，让下次回看能快速复现“可复制的动力”。

## 写什么？（引导提示）

当不知道写什么时，任选一句开头：

- 我现在的身体感觉是…
- 此刻让我动容/烦躁的触发点是…
- 如果只允许一句话记录今天，它会是…
- 我想留住的画面/声音/气味是…
- 我在逃避的其实是…
- 下一步我只做一件小事，它是…

保持不超过 100 字，专注“感受 + 线索”。

## 示例条目（≤100 字）

- 今天加班路上突然安心：原来我介意的不是工作量，而是“没有说清边界”，决定明早先把边界写出来。
- 和他聊完心有余温：被认真倾听时，胸腔像有暖灯亮起，整个人慢慢沉下来。
- 训练后无力却很踏实：肌肉微颤，心里不再急，感觉自己在向“可控的节律”靠近。
- 这一小时被消息打碎：不是我缺少专注，而是“没给自己关门的许可”。

## 如何实现？

- 超轻交互：启动即写，无分类压力；限制长度（默认 100 字）迫使聚焦核心感受。逻辑见 `ENTRY_CHARACTER_LIMIT` 与输入监听，参见 [`MemoWindow`](main.py)。
- 安全落档：文本和情绪经 [`escape_entry_for_json`](main.py) 处理后由 [`append_entry_to_journal`](main.py) 附带情绪、唯一 `id` 与时间戳写入 [`JOURNAL_PATH`](main.py)（默认 `journal.json`）。  

  ```json
  {
    "moments": [
      {
        "id": 1731164149,
        "timestamp": "2025-11-09T22:35:49+08:00",
        "mood": "calm",
        "text": "轻盈而笃定"
      }
    ]
  }
  ```

- 不打扰的提醒：窗口最小化到系统托盘；周期提醒（[`GENTLE_REMINDER_INTERVAL_MS`](main.py) 默认 1000ms 示例值，可调）唤起“记录当下”意识。
- 本地纯文本：无需账号，无网络依赖，降低遗失与泄露风险。
- 入口函数 [`main`](main.py) 负责应用生命周期；托盘逻辑与最小化行为由 `MemoWindow` 内部方法统一管理。

## 设计取舍

- 只保留“文本 + 时间”，刻意不加标签/文件夹：减少“组织成本”，守住零门槛。
- 有意限制长度：不是反生产力，而是为“抓住核心”服务；长文可以在回顾时再扩写。
- 轻提醒而非强打断：提醒的目的是“唤起自感知”，不是“逼你写作”。

## 如何使用？

### 安装

```bash
# 需要 Python >=3.13
pip install pyside6
python main.py
```

Windows 也可使用：

```bash
py -3.13 -m pip install pyside6
py -3.13 main.py
```

### 使用步骤

1. 启动后先在“情绪 Mood”下拉框中选定当前的情绪基调。
2. 在输入框里浓缩触动来源、身体感受或突发的动力念头。
3. 观察右下角计数，保持精炼（有意的长度限制帮助萃取“情感核心”）。
4. 点击“Archive to json”立即归档至 `data.json`。
5. 关闭窗口时程序进入托盘并定期提示“别忘了记录”；需要时单击托盘图标恢复。

## 常见问题

- 想写长一点怎么办？  
  先保存“核心 1 句”，长文可在回顾阶段另起文档扩写。
- 如何调整 100 字上限？  
  在 `main.py` 中修改 `ENTRY_CHARACTER_LIMIT`，并重启程序。
- 提醒太频繁/太稀疏？  
  在 `main.py` 中调整 `GENTLE_REMINDER_INTERVAL_MS`（毫秒）。
- 数据存哪儿？  
  默认 `data.json`（见 `JOURNAL_PATH`）。建议定期复制到你的同步盘或手动备份。
- 想自定义情绪选项？  
  在 `main.py` 中调整 `MOOD_CHOICES` 列表，可同时修改显示文案与存储值。
- 多设备同步？  
  目前不内置云同步，可用你自己的网盘/版本控制托管该 json 文件。

## 隐私与数据

- 数据以纯文本 json 保存在本地，便于备份、迁移与后续分析（如导入到数据工具）。
- 不采集、不上传——你的情绪只属于你。

## 回望与复盘（可选）

每周或每月，挑一个固定时段滚动浏览 `data.json`，回答三件事：

- 哪些瞬间最有能量？它们的共性是什么？
- 哪些反复出现的“刺痛点”在提醒我设定边界或做出调整？
- 我愿意在下一阶段保留或放下什么？

把“短句”串起来，你会看到一张专属的情感等高线图。
